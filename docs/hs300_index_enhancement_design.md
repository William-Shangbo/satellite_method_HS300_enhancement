# 沪深300指数增强策略设计

## 0. 目标与主线

我们要做的是相对沪深300的指数增强，而不是绝对收益择时策略。核心问题是：在沪深300成分股内，保留和基准相近的行业、市值、个股权重与波动暴露，同时用可解释、可复验的因子预测股票横截面收益，形成稳定的主动权重。

一句话假设：

> 沪深300成分股内部存在价值、质量、成长、动量反转、低波和流动性等横截面溢价；在严格控制相对基准风险后，这些溢价可以转化为稳定超额收益。

## 1. Universe 与数据频率

### 1.1 选股 universe

基础 universe 是 Tushare `index_weight` 中 `index_code=399300.SZ` 的沪深300成分股和权重。每个调仓日只在当期可见的成分股里选股，避免幸存者偏差。若未来改成中证口径，也可以切到 `000300.SH`，但当前平台配置使用 `399300.SZ`。

交易约束在 universe 上再叠一层：

- 剔除当日不可交易股票：停牌、上市不足一定天数、ST 或退市状态。
- 剔除涨跌停导致无法买卖的股票，至少在回测成交模型中做保守处理。
- 成分权重缺失或异常时，以最近一期有效指数权重前向填充，但不得越过调仓可见日期。

Tushare 需要下载的数据：

- `stock_basic`：证券主表、上市状态、上市/退市日期。
- `trade_cal`：交易日历和月末调仓日期。
- `index_weight`：沪深300成分和基准权重。
- `daily` + `adj_factor`：复权价格、收益、动量、波动率。
- `daily_basic`：估值、市值、换手率、股息率。
- `income`、`balancesheet`、`cashflow`、`fina_indicator`：财务、质量、成长、杠杆类因子。
- 可选：`suspend_d`、`limit_list`、`namechange` 做可交易性和历史名称过滤。

### 1.2 调仓频率与数据频率

底层数据采用日频，因子截面采用月频。调仓日在每月最后一个交易日收盘后形成持仓，下一交易日执行，持有到下一个调仓日。财务数据必须按 `ann_date` 做 point-in-time 对齐，只能使用调仓日前已经公告的数据。

建议先用月频作为主版本，因为它和原 `HS300_index_enhance` repo 的框架一致，也能降低换手和交易成本。等月频版本跑通，再评估双周频或周频是否带来更好的信息比率。

## 2. 如何创造超额

### 2.1 因子挑选、构造与组合

因子分三层推进。

第一层是基础可解释因子：

- 价值：`EP=1/pe_ttm`、`BP=1/pb`、`SP=1/ps_ttm`、股息率。
- 成长：营收/净利润同比、ROE/ROA 变化。
- 质量：ROE、ROA、毛利率、净利率、经营现金流质量、资产周转。
- 动量反转：1/3/6/12 月收益，短期反转和中期动量分开。
- 波动率：20/60/120 日收益波动、残差波动。
- 流动性：换手率、成交额、换手偏离。
- 市值：总市值、流通市值、非线性市值。

第二层是单因子检验：

- 横截面 IC / Rank IC：看方向、均值、IR、胜率。
- 分层回测：看单调性、Top-Bottom spread 和分年稳定性。
- 稳健性：换窗口、换中性化方法、加交易成本、分市场阶段。
- 相关性：高相关因子只保留一个代表，避免重复押注。

第三层是因子组合：

- 每期对单因子做缺失处理、去极值、标准化。
- 对行业和市值做中性化，减少风格污染。
- 按 ICIR、分层 spread、稳定性给因子打分。
- 因子合成可先用等权，再升级为滚动 ICIR 加权或 ridge/elastic-net 预测模型。

### 2.2 组合优化

优化目标是最大化预测主动收益：

`max active_weight dot expected_active_return`

同时满足：

- 权重和为 1。
- 个股权重非负，初版不做卖空。
- 个股主动偏离上限，例如 `|w_i - w_b_i| <= 1%`。
- 单票权重上限，例如 `w_i <= 3%`。
- 行业主动暴露上限，例如每个行业相对基准偏离不超过 `3%`。
- 预估跟踪误差上限，例如年化 `4%-6%`。
- 换手率上限，例如单次换手不超过 `30%-50%`。

当前实现先放一个无 SciPy 依赖的可解释启发式优化器：按 alpha score 生成主动权重，再逐步裁剪个股偏离、非负、单票上限和换手。等数据和验证稳定后，可以接入二次规划求解器做严格风险模型优化。

## 3. 风控

### 3.1 控制回撤

回撤控制分为组合内控制和组合外控制。

组合内：

- 控制个股、行业、市值暴露，避免某个风险源主导组合。
- 控制换手和流动性，减少交易冲击。
- 因子组合中避免高相关拥挤因子。

组合外：

- 监控主动净值最大回撤。
- 若主动回撤超过阈值，例如 `-5%`，降低主动权重预算。
- 若最近 3-6 个月 IC 失效，降低对应因子权重或暂停。

### 3.2 控制 tracking error

事前 tracking error 由主动权重和风险模型估计，先用日收益协方差近似，后续接 Barra 风险模型或行业+风格协方差。

事后 tracking error 用日度超额收益计算：

`TE = std(portfolio_return - benchmark_return) * sqrt(252)`

组合上线前必须同时展示：

- 年化超额收益。
- 年化 tracking error。
- 信息比率 IR。
- 最大主动回撤。
- 月度胜率和分年度表现。

## 5. 第一版落地范围

第一版目标不是“最优收益”，而是跑通闭环：

1. 用 Tushare 下载沪深300成分、权重、日行情、daily_basic、财务指标。
2. 构造月末截面，基础字段对齐原 `HS300_index_enhance` 的 schema。
3. 先实现价值、质量、动量、波动、流动性、市值六类基础因子。
4. 用 IC/分层筛选因子，形成 composite alpha。
5. 用启发式主动权重优化器构建组合。
6. 输出 HTML 报告和 CSV 指标表。

第二版再加入更严格的财务 point-in-time、行业中性、二次规划、交易成本模型和 walk-forward 验证。
